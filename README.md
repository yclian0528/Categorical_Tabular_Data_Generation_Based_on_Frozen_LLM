# A Conditional Framework for Categorical Tabular Data Generation Based on Frozen Large Language Models for Semantic Feature Extraction

This repository provides the core implementation, public datasets, example generated data, and evaluation workflow used to reproduce the main experimental results of the paper.

The proposed method is a conditional framework for categorical tabular data generation. It does not replace existing tabular generators. Instead, it takes synthetic numerical fields generated by an existing baseline model as conditional input, uses a frozen large language model backbone to extract semantic representations from the numerical conditions, and reformulates categorical field generation as a multi-task classification problem over valid category sets. The framework can be used as a post-processing enhancement module for existing tabular generation methods, improving categorical field quality, output validity, and numerical-categorical conditional consistency.

Large pretrained backbones and trained checkpoints are not included in this repository due to file size. This repository includes the core implementation, datasets, generated baseline/enhanced CSV files, and evaluation code for reproducing the main results.

## Usage Scenarios

### A. Reproduce Main Paper Results

If you only want to reproduce the main experimental metrics, use the provided data and evaluation notebook under `demo/`. This workflow does not require retraining, pretrained LLM backbones, or trained checkpoints.

The `demo/` directory includes:

- `baseline_data/`: synthetic CSV files generated by Gaussian Copula, CTGAN, TVAE, GReaT, and TabDDPM.
- `enhanced_data/`: enhanced CSV files where baseline numerical fields are kept and categorical fields are regenerated by the proposed method.
- `origin_train_dataset/` and `origin_test_dataset/`: real train/test splits for each random seed.
- `fidelity&utility_metric.ipynb`: an example notebook for computing statistical fidelity and downstream utility metrics.

Recommended workflow:

```bash
cd demo
jupyter notebook "fidelity&utility_metric.ipynb"
```

The notebook reads baseline synthetic data, enhanced synthetic data, and real train/test splits from `demo/`, then computes the main evaluation metrics. The main experiments compare native baseline outputs with enhanced outputs in terms of statistical fidelity and TSTR downstream utility. Since the full set of experiment combinations and outputs is large, this repository provides a representative subset as an example reproducibility package.

### B. Train from Scratch and Generate Enhanced Synthetic Data

Use `core_code/` if you want to run the full core pipeline. You need to prepare:

- Original dataset CSV files for training.
- Synthetic CSV files to be enhanced.
- Offline pretrained LLM backbones, such as Hugging Face-format `gpt2`, `bert-base-uncased`, or other model directories.
- Dataset names, backbone names, baseline model names, paths, and training settings in `core_code/src/config/config.yaml`.

## Environment Setup

Core dependencies are listed in `core_code/requirements.txt`. Python 3.10+ and a CUDA GPU are recommended.

```bash
cd core_code
pip install -r requirements.txt
```

`requirements.txt` includes core dependencies such as PyTorch, Transformers, pandas, scikit-learn, and PyYAML. The listed PyTorch packages target CUDA 12.1; if your CUDA version differs, adjust the `torch`, `torchvision`, and `torchaudio` installation versions accordingly.

## Configuration

The core pipeline is controlled by `core_code/src/config/config.yaml`. Common settings include:

- `project.seeds`: random seeds for experiments.
- `experiment_targets.datasets`: dataset names to process; each must correspond to `{paths.raw_data_dir}/{dataset}.csv`.
- `experiment_targets.backbones`: frozen LLM backbone names; each must correspond to `{paths.offline_models_dir}/{backbone_name}`.
- `experiment_targets.num_gen_models`: baseline synthetic data generator names.
- `paths.raw_data_dir`: directory for original dataset CSV files.
- `paths.offline_models_dir`: directory for offline pretrained backbones.
- `paths.syn_raw_source_root`: source directory for baseline synthetic CSV files.
- `paths.final_syn_output`: output directory for final enhanced CSV files.

## Core Pipeline

Run the following commands from the `core_code/` directory.

### Full Pipeline

Before running, make sure that datasets, backbones, baseline synthetic data paths, and output paths are correctly configured in `core_code/src/config/config.yaml`.

```bash
cd core_code

python preprocess_training_data.py
python build_semantic_embeddings.py
python extract_numeric_features.py
python train_feature_classifier.py
python preprocess_synthetic_data.py
python generate_synthetic_categories.py
```

### 1. Preprocess Original Training Data

```bash
python preprocess_training_data.py
```

This step reads the raw CSV files specified in `config.yaml`, creates seed-specific train/validation splits, and outputs processed data and label encoder artifacts for later training.

### 2. Build Semantic Embeddings for Categorical Labels

```bash
python build_semantic_embeddings.py
```

This step uses an offline frozen LLM backbone to convert candidate category values in each categorical field into semantic prototypes. The outputs are saved under the processed data artifact directories.

### 3. Extract Numerical Condition Representations

```bash
python extract_numeric_features.py
```

This step serializes numerical fields into text descriptions, feeds them into a frozen LLM backbone, and extracts conditional representations for the multi-task categorical generation model.

### 4. Train the Categorical Field Generation Model

```bash
python train_feature_classifier.py
```

This step trains the multi-task categorical classification model. Model components such as MMoE, normalized numerical feature concatenation, and Cosine Head are controlled by `config.yaml`. Training outputs, checkpoints, and experiment settings are saved to the configured experiment output directory.

### 5. Preprocess Baseline Synthetic Data

```bash
python preprocess_synthetic_data.py
```

This step reads complete baseline synthetic CSV files, keeps their numerical fields, and prepares inference inputs for categorical generation.

### 6. Generate Enhanced Synthetic Data

```bash
python generate_synthetic_categories.py
```

This step keeps the numerical fields from baseline synthetic data, regenerates categorical fields with the trained model, and outputs complete enhanced synthetic CSV files.

- CSV filenames match `experiment_targets.datasets` in `config.yaml`.
- Each dataset contains both numerical and categorical fields.
- The target field required for TSTR evaluation exists in the data.
- Baseline synthetic CSV files use column names consistent with the original data and include the numerical fields used as conditional input.
